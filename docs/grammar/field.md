# 局部域

***

> **定义域**，**值域**和**局部域**一直是我们的研究对象。

## 什么是域？

首先，如果想要知道局部域是什么，我们就需要知道，在一个编程语言中，域是是什么。

**域**(scope)，顾名思义，就是一块地方(不要看我文件名是field就觉得域的英文是field啊！)，它通常表现出极强的分布性，就像生物群落的水平分布一样。

我们需要知道的是，局部域中可以存在另一个局部域，这是一种嵌套关系。需要指出的是，局部域的嵌套并不是什么奇怪的情况，因为这发生在我们的周围的方方面面。

我们一般把一个由花括号包起来的内容称为"域"，这个部分就形成了一个与外界分隔开的专属小世界，就像细胞一样，花括号就是它的细胞膜。在一个局部域中，是无法与它并列的局部域内发生联系的。我们把不同的域中的对象/示例都单独给了一个分类，这个分类叫做**名称空间**(namespace)。

所以你会发现我在主页面编写的情况：

```javascript
{
    var a = 114;
    print(a);
    {
        var a = 514;
        print(a);
    }
    print(a);
}
```

为什么三次输出不一样呢？这里面的原理是这样的：首先，你在你在`a域`中创建了变量`a`，然后输出变量`a`；然后你在`a域`里的`b域`里创建了另一个变量`a`，然后输出了另一个`a`。第二个`a`和第一个`a`是两个不同的变量！

我们有一个很牛逼的东西叫做名称空间成员访问符，它的样子很可爱，也很容易写，长这样`::`。通过它，你可以获取某个特定名称空间里的内容。和成员访问符不同的是，成员访问符是访问一个大的`集合`里的内容，这个集合被一个“别名”指代，比如一个导入的库里的导出函数，或者一个类的实例里的成员函数。但是名称空间与它们不同。名称空间相较于“别名”，它更像是个ID。每个域都应该有一个独一无二的ID。

举个例子，我们再来看上面的代码，在解释器看来，它大概长这样：

```js
{
    namespace frist {
        var a = 114;
    }
    print(frist::a);
    {
        namespace second {
            var a = 514;
        }
        print(second::a);
    }
    print(frist::a);
}
```

可以看到，两个不同的变量在定义的时候拥有不同的名称空间，它们有一个独特的ID，所以解释器在处理的时候会将他们视为两个不同的变量。而一个域的名称空间会被随机分配，这里只是被我人为的写好看了。

调用一个域里的对象或函数，会优先选择调用这个域里定义的对象或函数。如果这个域里没有定义这个对象或函数，就会调用父域里的对象或函数。如果你想要在这个域调用它父域的内容，那么你需要添加访问符，也就是`::`。

在这里，我们引入一个新的语法，那就是规定名称空间：

```js
namespace <name> {<definition>}
```

花括号中的内容将带有你自己定义的名称空间ID了。也就是说，你可以完成这样的操作：

```js
namespace awa {
    def amns(a) {
        print(a);
        return a;
    }
}

namespace qwq {
    def amns(a) {
        print(a + "is" + a);
    }
}

qwq::amns(awa::amns(114));
```

两个函数名完全一样，但是它们有不同的ID，所以可以同时存在。

## 什么是局部域？

**局部域**指的是用户在任何位置创建的域，也就是那些花括号包起来的部分。

也就是说，你创建的每一个函数，创建的每一个判断语句，创建的每一个循环甚至是你创建的每一个类，都是一个局部域。

在局部域中，存在各种有意思的内容，比如我在[分支](./conditions)就提到了你可以使用`break`退出这个域，也许这可以作为一个新的循环写法？~~(虽然这很不合适就是了)~~

## 生命周期和作用域

还有一个很有意思的东西，那就是变量在局部域中的管理。

你要知道，创建一个变量就会腾出来一块内存存储它。但是如果我已经用完了这个变量，不需要它了，怎么办？这就要提到变量的销毁了。

我们在一个域里面创建变量，然后在域里面使用它。但是当我们在这个域里面退出了，变量就会被销毁，内存就会被释放，让后面的执行重新获得内存。

而变量创建到变量销毁的过程就被称为变量的**生命周期**。所有需要创建的新对象以及会在运行过程中销毁的对象都会有属于它的生命周期。这个过程是由解释器自己执行的，我们不需要管它。

而**作用域**则是这个对象可以被使用的地方，比如下面的一个很有意思的错误示例：

```js
{var i = 1;}
{print(i);}
```

我们把这种写法称为"异域症"，因为你在一个和创建变量毫无关系的域里面调用了这个变量。后者完全不能访问到这个变量，因为前者创建的变量只能在前一个域中存在，正如前面说的，这个域一旦结束，所有内部创建的对象都会被销毁以换取内存。所以离开这个域之后，你就会发现变量消失了，与其相关的所有东西都消失了，你调用的东西不存在，报错了！就像你女朋友跟你分手去找其他人但是你忽然想找人聊天却没有人在这里！

简单来讲，某个对象的作用域指的是程序创建该对象的域。它只会在这个域中存在，所以其他的域就无法调用这个对象。这是个很严肃的问题，本语言会在发现这样的情况的时候提醒你未定义的对象。

## 全局域

需要被提及的是，你在任何一个不是你创建的域里(也就是在你写的所有花括号之外)创建的内容会被归属为**全局域**内容。我们一般称之为`global`，也就是说，如果你在`main()`函数之外的文件头里创建了一个变量`a`，它的全称是`global::a`，但是为了方便，我们一般写作`::a`。所以你不能规定一个域的名称空间为`global`。

全局域的作用域是所有地方都可以用。不管是你的函数内，还是某个域内，或者是另一个文件。当然，我们并不建议你把什么东西在全局域创建，因为全局域里的所有东西，都会在程序执行结束的时候被销毁。我们不保证数据的合法性，数据的大小等，是否会让你的电脑爆炸。
