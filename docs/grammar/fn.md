## 函数相关

- `def <func> (<value>) {code}` 定义一个名为`<func> `的函数，并需要传入参数`<value> `

参数应该在传入和调用之前定义其类型；传入的时候，是一次拷贝，将值拷贝到参数变量中，你对它的任何修改都不会影响到原来的内容。

若函数执行到`return`，则返回后面的内容，如果执行结束还没有`return`，则返回一个空值`NULL`。

如果`return`其后不跟随任何内容，返回依旧是空值，这是被允许的，这通常标记着函数的结尾。

函数的参数可以是多个，比如`def <func> (<value1>,<value2>) {code}`。值得一提的是，你可以如此编写函数：

```javascript
def test(var a = 1,var b = 2,var c = [1,2,3]) {
    // 代码内容
}
```

这样就会给参数设置默认值，这个默认值是在调用函数未传入参数的时候生效。如果你不填某个参数，那么它将传入默认值，比如你可以直接调用`test()`，也可以调用`test(123)`，这将默认使用`b = 2`以及`c = [1,2,3]`的设置。需要注意的是，空参数不能在实参中间，比如`test(1,,[1,2])`，这是错误的！

需要注意的是，参数的创建是个拷贝的过程，它会新建一个变量，并且在调用函数的时候求值。

## 关于返回值

返回值`return`指的是一个块在执行之后返回的结果，这里我们通常认为每一个代码块都可以拥有一个`return`来返回值或者停止运行(跳出代码块)。因为在本语言中，函数创建类似于一个定义或替换的过程：

```javascript
def test(var a) {
    print(a,"\t");
    return;
}
```

在这个函数定义中，我们类似为后面的代码块`{print(a,"\t");return;}`设置了一个别名`test(a)`。所以实际上，我们在调用函数的时候，我们只是在复制一份代码块过来。所以普通的代码块也可以拥有返回值，就像下面那样：

```javascript
var a = input("Type a number: ");
var b = {print("Let me thinking...");return 5**a;};
print("The 5 ^ " + a + "is" + b);
```

输入输出大概就是：

```text
Type a number: 3
Let me thinking...
The 5 ^ 3 is 125
```

所以我们允许独立的代码块中拥有`return`语句，这是极为重要的一点。
